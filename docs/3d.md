# OpenFPGA 3D Accelerator

## 1. Architecture

A fixed-function 3D rasterizer with a **triangle pipeline + span fast-path**. The CPU handles all 3D transforms and submits screen-space geometry via a ring buffer in SDRAM. The GPU rasterizes, textures, lights, depth-tests, and writes pixels to the framebuffer in parallel.

```
    CPU (VexRiscv @ 100 MHz)
         |
    [SDRAM Command Ring Buffer]      ← CPU writes commands here
         |
    ┌────┴─────────────────────────────────────────────────────┐
    │  GPU (~100 MHz, 5K ALMs, 64 M10K, 6 DSP)                │
    │                                                           │
    │  [DMA Fetch] → [Cmd Decode] → [State Registers]          │
    │                      |                                    │
    │              ┌───────┴───────┐                            │
    │              │               │                            │
    │       [Triangle Setup]  [Span Bypass]                     │
    │       (edge equations)  (direct params)                   │
    │              │               │                            │
    │              └───────┬───────┘                            │
    │                      │                                    │
    │              [Scanline Rasterizer]                         │
    │              (walk edges, emit fragments)                  │
    │                      │                                    │
    │              [Fragment Processor]                          │
    │              ├─ Texture Cache (M10K)                       │
    │              ├─ Colormap BRAM (M10K)                       │
    │              ├─ Color interpolation                        │
    │              └─ Alpha test                                 │
    │                      │                                    │
    │              [Depth Test] ←→ SRAM Z-buffer                 │
    │                      │                                    │
    │              [FB Writer] → AXI4 → SDRAM framebuffer        │
    └───────────────────────────────────────────────────────────┘
```

### 1.1. Resource Budget

| Component               | ALMs  | M10K | DSP | Notes                                    |
|--------------------------|-------|------|-----|------------------------------------------|
| DMA command fetch        | 300   | 4    | 0   | AXI4 read, ring pointer management       |
| Command decoder + state  | 300   | 2    | 0   | Unpack commands, sticky state registers   |
| Triangle setup           | 1000  | 4    | 4   | Edge equations, attribute gradients       |
| Scanline rasterizer      | 700   | 4    | 0   | Edge walking, fragment emission           |
| Span bypass path         | 150   | 0    | 0   | Mux span params into rasterizer          |
| Fragment processor       | 1000  | 0    | 2   | Texcoord step, color interp, alpha test  |
| Texture cache            | 300   | 20   | 0   | 4-way set-assoc, 16B lines, AXI4 fill   |
| Colormap BRAM            | 100   | 16   | 0   | 16KB: 64 light × 256 palette             |
| Z-buffer interface       | 400   | 2    | 0   | SRAM read/compare/write pipeline          |
| Framebuffer writer       | 350   | 4    | 0   | 4-byte accumulator, AXI4 burst writes    |
| Misc (FIFO, fence, perf) | 300   | 4    | 0   | Fence counter, stat counters             |
| **Total**                | **4900** | **60** | **6** |                                     |
| **Budget**               | **5000** | **64** | **~59** |                                   |

Headroom: ~100 ALMs, 4 M10K.

### 1.2. Command Ring Buffer

64KB ring in SDRAM. CPU writes commands sequentially, updates `wr_ptr` on `kick()`. GPU DMA reads commands, advances `rd_ptr`. Fence commands write a sequence number the CPU can poll.

```
Word 0:  [31:24] cmd_type  [23:0] payload_words
Word 1+: command-specific payload
```

| Cmd    | Name              | Payload | Description                                |
|--------|-------------------|---------|--------------------------------------------|
| 0x01   | NOP               | 0       | Padding / alignment                        |
| 0x02   | FENCE             | 1       | Write fence_value to status reg            |
| 0x10   | CLEAR             | 2       | flags, color, depth                        |
| 0x20   | SET_TEXTURE       | 4       | addr, w, h, format, wrap                   |
| 0x21   | SET_DEPTH_FUNC    | 1       | depth function enum                        |
| 0x22   | SET_BLEND         | 1       | blend mode + alpha ref                     |
| 0x23   | SET_FRAMEBUFFER   | 2       | addr, stride                               |
| 0x24   | SET_ZBUFFER       | 2       | addr, stride                               |
| 0x25   | SET_SHADE_MODE    | 1       | gouraud enable                             |
| 0x30   | DRAW_TRIANGLES    | 1+N×6   | count, then N vertices (6 words each)      |
| 0x31   | DRAW_INDEXED      | 2+V×6+I | vert_count, idx_count, verts, indices      |
| 0x40   | DRAW_SPAN         | 18      | full span descriptor                       |
| 0x41   | DRAW_SPANS_BATCH  | 1+N×18  | count, then N span descriptors             |

Vertex encoding (6 words = 24 bytes):

```
Word 0: [31:16] x (12.4)    [15:0] y (12.4)
Word 1: [31:16] z (16-bit)  [15:0] pad
Word 2: s (16.16 fixed-point)
Word 3: t (16.16 fixed-point)
Word 4: w (16.16 fixed-point, 1/W for perspective, 0x10000 for affine)
Word 5: [31:24] a  [23:16] b  [15:8] g  [7:0] r
```

### 1.3. Triangle Pipeline

**Setup** (~15-25 cycles): computes edge equations and attribute gradients using 4 DSP blocks time-shared. Reciprocal via CLZ + LUT + DSP correction.

**Rasterizer** (~1 cycle/pixel): integer edge stepping (add A per X, add B per Y). 12.4 sub-pixel precision, top-left fill rules.

**Span bypass**: DRAW_SPAN skips setup, feeds pre-computed params directly into the fragment processor.

**Fragment processor** (~2-4 cycles/pixel): texture fetch → colormap/color combine → alpha test → depth test → FB write.

**Texture cache**: 20KB, 4-way set-assoc, 64 sets, 16B lines. Hit: 1 cycle. Miss: ~10-40 cycles (AXI4 fill).

**Colormap BRAM**: 16KB, 64×256 entries. Upload once at level load.

**Z-buffer**: 16-bit unsigned, external SRAM, 1-2 cycle latency.

### 1.4. Performance

At 100 MHz, 320×200 (64K pixels):

| Mode                          | Cycles/Px | Mpx/sec | FPS (fill) |
|-------------------------------|-----------|---------|------------|
| Span (Doom/Duke3D)            | 3-5       | 20-33   | 312-520    |
| Triangle + affine + I8/cmap   | 4-5       | 20-25   | 312-390    |
| Triangle + affine + RGB565    | 4-5       | 20-25   | 312-390    |
| Span + perspective (Quake)    | 8-12      | 8-12    | 130-190    |
| Triangle + perspective + I8   | 8-12      | 8-12    | 130-190    |
| Clear (DMA)                   | 0.25      | 400     | N/A        |

Fill rate is never the bottleneck at 320×200. Even perspective mode fills the screen 2-3× per frame at 30 FPS. The bottleneck is CPU-side transforms and game logic.

## 2. Implementation Phases

### Phase 1: Span Engine — `[ ]`

Build the core pipeline without triangle setup. Targets Doom and Duke3D.

**Hardware:**
- [ ] Command ring buffer + DMA fetch engine
- [ ] Command decoder + state registers
- [ ] Span rasterizer (stepping engine)
- [ ] Texture cache + colormap BRAM
- [ ] Framebuffer writer (AXI4 burst)
- [ ] Z-buffer interface (SRAM)
- [ ] Fence mechanism + stat counters

**Software (of_gpu.h):**
- [ ] `of_gpu_init` / `of_gpu_shutdown`
- [ ] `of_gpu_colormap_upload`
- [ ] `of_gpu_set_framebuffer` / `of_gpu_set_zbuffer`
- [ ] `of_gpu_clear`
- [ ] `of_gpu_draw_span`
- [ ] `of_gpu_submit` / `of_gpu_wait` / `of_gpu_finish`
- [ ] Ring overflow protection (spin-wait when full)

**Validates:** async model, texture cache, memory interfaces, pixel pipeline.

**Enables:** Doom, Duke3D (full acceleration), Quake (span path).

---

### Phase 2: Triangle Setup — `[ ]`

Add triangle rasterization. Targets SM64, Descent, Wipeout.

**Hardware:**
- [ ] Edge equation computation (sequential DSP reuse)
- [ ] Attribute gradient computation (Z, S, T, W, R)
- [ ] Scanline rasterizer (edge walking, fragment emission)
- [ ] Sub-pixel precision (12.4) + top-left fill rules
- [ ] Feed fragments into existing fragment processor

**Software (of_gpu.h):**
- [ ] `of_gpu_bind_texture`
- [ ] `of_gpu_depth_test`
- [ ] `of_gpu_blend`
- [ ] `of_gpu_shade_mode` (gouraud)
- [ ] `of_gpu_draw_triangles`

**Validates:** triangle rendering, sub-pixel accuracy, fill rules.

**Enables:** SM64, Descent, Wipeout, Quake 2 / Half-Life (triangle path).

---

### Phase 3: OpenGL 1.x Core — `[ ]`

CPU-side GL state machine + minimal hardware additions to support GLQuake, Quake 2, Quake 3, Half-Life.

#### 3a. CPU-Side GL State Machine — `[ ]`

All software, no hardware changes.

- [ ] Matrix stacks: `GL_MODELVIEW` (32 deep), `GL_PROJECTION` (2 deep), `GL_TEXTURE` (2 deep)
- [ ] Matrix ops: `LoadIdentity`, `MultMatrix`, `Translate`, `Rotate`, `Scale`, `Frustum`, `Ortho`
- [ ] Vertex transform: multiply by MVP, perspective divide
- [ ] Viewport mapping: NDC → screen coords (`glViewport`, `glDepthRange`)
- [ ] Frustum culling: trivial accept/reject (6-plane dot product)
- [ ] Near-plane clipping: Sutherland-Hodgman (mandatory — prevents W=0 crash)
- [ ] Backface culling: `glCullFace` / `glFrontFace` (post-transform cross product)
- [ ] Primitive decomposition: `GL_QUADS` → 2 triangles, `GL_POLYGON` → fan, `GL_TRIANGLE_FAN` → triangles

#### 3b. Scissor Test — `[ ]`

~50 ALMs. Simple bounds check in the rasterizer.

- [ ] 4 registers: x_min, x_max, y_min, y_max
- [ ] Per-fragment compare + discard
- [ ] `SET_SCISSOR` ring command
- [ ] `of_gpu_scissor(x, y, w, h)` API

#### 3c. Depth Write Mask — `[ ]`

~10 ALMs. Required for transparent surfaces (draw opaque with Z write, then transparent with Z test only).

- [ ] 1-bit state register, gate on Z-buffer write
- [ ] `SET_DEPTH_MASK` ring command
- [ ] `of_gpu_depth_mask(bool write)` API

#### 3d. Full Alpha Test — `[ ]`

~30 ALMs. Current `of_gpu_alpha_ref` conflates alpha ref with blend mode.

- [ ] Separate `SET_ALPHA_FUNC` command: function enum + ref value
- [ ] Support: `NEVER`, `LESS`, `EQUAL`, `LEQUAL`, `GREATER`, `NOTEQUAL`, `GEQUAL`, `ALWAYS`
- [ ] `of_gpu_alpha_func(func, ref)` API

#### 3e. Triangle Strips — `[ ]`

~100 ALMs (or CPU-side decomposition at zero ALM cost).

Hardware option:
- [ ] `DRAW_TRIANGLE_STRIP` command: reuses last 2 vertices, alternates winding
- [ ] `DRAW_TRIANGLE_FAN` command: reuses first + last vertex

Software option (no hardware change):
- [ ] CPU decomposes strips/fans into `GL_TRIANGLES` before submission
- [ ] 3× vertex bandwidth cost but zero ALM cost

#### 3f. Vertex Fog — `[ ]`

CPU-side only. No hardware change needed if fog color is baked into vertex color.

- [ ] Per-vertex fog factor: `f = (end - z) / (end - start)` (linear), or exp/exp2
- [ ] Blend: `color = f * vertex_color + (1-f) * fog_color`
- [ ] Encode result into vertex R/G/B before submission

Limitation: per-vertex fog only, no per-pixel. Acceptable for target games.

#### 3g. Per-Vertex Lighting — `[ ]`

CPU-side only.

- [ ] 2-light minimum (most retro GL games use 0-1 dynamic lights)
- [ ] Ambient + diffuse + specular per light
- [ ] Positional / directional
- [ ] Attenuation (constant / linear / quadratic)
- [ ] Compute per-vertex, encode into vertex color

---

### Phase 4: Framebuffer Blending — `[ ]`

~400 ALMs. Requires adding a read path to the framebuffer writer (read-modify-write).

- [ ] FB read-back path (AXI4 read before write)
- [ ] Blend equation: `src * srcFactor + dst * dstFactor`
- [ ] `SET_BLEND_FUNC` ring command with src/dst factor enums
- [ ] Required factors:
  - [ ] `(GL_ONE, GL_ZERO)` — opaque (already works: no blend)
  - [ ] `(GL_SRC_ALPHA, GL_ONE_MINUS_SRC_ALPHA)` — transparency
  - [ ] `(GL_ONE, GL_ONE)` — additive (particles, explosions)
  - [ ] `(GL_DST_COLOR, GL_ZERO)` — modulate (lightmaps)
  - [ ] `(GL_ZERO, GL_SRC_COLOR)` — modulate (alt form)
  - [ ] `(GL_SRC_ALPHA, GL_ONE)` — additive transparency
- [ ] `of_gpu_blend_func(src, dst)` API

**Enables:** translucent walls (Duke3D), lightmap passes (Quake 2/3), particle effects.

---

### Phase 5: Texture Quality — `[ ]`

#### 5a. Bilinear Filtering — `[ ]`

~300 ALMs, +2 M10K. 4 texel reads per fragment instead of 1.

- [ ] Fractional S/T bits select blend weights
- [ ] 2×2 texel fetch from cache
- [ ] Bilinear interpolation (2 DSP multiplies)
- [ ] `SET_TEXTURE_FILTER` command (nearest / bilinear)
- [ ] `of_gpu_texture_filter(mode)` API

#### 5b. Mipmapping — `[ ]`

~200 ALMs, +1 M10K.

- [ ] LOD calculation from texture coordinate derivatives
- [ ] Mip chain base address per level (stored in texture state)
- [ ] `GL_NEAREST_MIPMAP_NEAREST` (select nearest mip, point sample)
- [ ] `GL_LINEAR_MIPMAP_NEAREST` (select nearest mip, bilinear sample)
- [ ] Trilinear (`GL_LINEAR_MIPMAP_LINEAR`) deferred — requires 8 texel reads

#### 5c. Perspective Correction — `[ ]`

~400 ALMs, +1 M10K.

- [ ] Per-16px subdivision (Quake quality)
- [ ] Interpolate S/W, T/W, 1/W, then divide at subdivision points
- [ ] Reciprocal unit shared with triangle setup

---

### Phase 6: Indexed Drawing — `[ ]`

Vertex reuse for meshes. Significant bandwidth reduction for SM64, Quake 2, Half-Life.

- [ ] `DRAW_INDEXED` ring command: vertex array + index list
- [ ] 16-bit indices
- [ ] Small vertex cache (8-16 entries, post-transform)
- [ ] `of_gpu_draw_indexed(verts, vert_count, indices, idx_count)` API

---

## 4. API Bugs to Fix

Track these before building hardware.

- [ ] `of_gpu_colormap_upload`: unused `src32` variable; byte-at-a-time upload (use word upload for 4× speed)
- [ ] `of_gpu_draw_span`: header says 18 payload words, arch doc says 16 — reconcile with hardware
- [ ] `of_gpu_draw_triangles`: `count` parameter is vertex count but name implies triangle count — clarify or rename
- [ ] `of_gpu_alpha_ref`: conflates alpha ref with blend enable — separate into `SET_ALPHA_FUNC`
- [ ] `of_gpu_shutdown`: calls `of_gpu_submit` before it is defined (works due to inline, but fragile)
- [ ] Ring overflow: `_gpu_ring_write` has no overflow check — add spin-wait in `_gpu_cmd_header`
- [ ] Static mutable state in header: will break if included from multiple translation units
- [ ] gpudemo: race condition writing `checkerboard_tex[0]` while GPU may still be reading it
